# Скриншоты для README

Создайте эти четыре файла в этой папке. Рекомендуется ширина 1200-1400 px, PNG, обрезанные до содержательной части.

## Список

### 1. `home.png` — главная страница

Откройте http://localhost:5173/ в развернутом окне браузера. Должно быть видно:
- Заголовок «Transcriber» сверху
- Три вкладки: «Папка», «YouTube», «Загрузить файл»
- Активная вкладка «Папка» с заполненным путём (например, `C:/Courses/Python`)
- Секция «История» снизу с 3-5 задачами в разных статусах (обработано / в работе / готово)

**Совет:** перед скриншотом запустите батч на небольшую папку — так в истории будет разный статус.

### 2. `folder-picker.png` — модалка выбора папки

На вкладке «Папка» нажмите «Обзор…». Откроется модалка с навигацией. Снимок:
- Модалка в фокусе, фон затемнён
- Открыт список дисков или содержимое какой-то папки
- Видна строка пути сверху модалки

### 3. `task-page.png` — страница задачи

Откройте любую завершённую задачу (`completed`). Должно быть видно:
- Заголовок с именем файла и бейдж «готово»
- Блок метаданных (язык, длительность, модель)
- Аудиоплеер
- Панель с кнопками экспорта
- Список сегментов с таймкодами, один — подсвечен синим (как будто сейчас воспроизводится)

**Совет:** нажмите play, дождитесь, пока плеер окажется в середине одного из сегментов, потом сделайте скриншот.

### 4. `search.png` — полнотекстовый поиск

На главной переключите режим поиска на «по тексту», введите слово, встречающееся в каких-то транскриптах (например, «стратегия», «модель», «пример»). В скриншот должны попасть:
- Переключатель «по тексту»
- Строка с введённым словом
- Список найденных задач
- Панель фильтров (можно скрыть, если мешает)

## Как сделать скриншот на Windows

- **Win + Shift + S** — встроенный инструмент выделения области
- **Win + PrtSc** — весь экран, сохранится в `Изображения\Снимки экрана`

## Оптимизация размера

PNG может весить много. Для README лучше 200-500 KB на картинку. Если тяжёлые:
- https://tinypng.com/ — оптимизация без видимой потери качества
- Или конвертация в `.webp` — но не все вьюверы GitHub его покажут в preview
